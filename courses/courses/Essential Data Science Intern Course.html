1Ô∏è‚É£ What is Data Science?

Data Science is an **interdisciplinary field** that uses scientific methods, processes, algorithms, and systems to extract **knowledge and insights** from structured and unstructured data. It is the core of modern decision-making.

  • Interdisciplinary ‚Äì It sits at the intersection of **Statistics**, **Computer Science**, and **Domain Expertise** (Business Knowledge).
  • Goal ‚Äì To turn raw data into actionable insights and predictions.

Role Primary Focus Key Toolset

3Ô∏è‚É£ Data Science Lifecycle Overview

The process follows a loop: Ask **Business Question** ‚Üí **Acquire** Data ‚Üí **Clean/Prepare** Data ‚Üí **Explore** Data (EDA) ‚Üí **Model** Data (ML) ‚Üí **Communicate** Results.

Page 2: Data Acquisition & Preparation (ETL)

1Ô∏è‚É£ The 80/20 Rule in Data Science

Data scientists typically spend **80% of their time** simply finding, cleaning, and organizing data, and only 20% on the actual analysis or modeling. This is the hardest part of the job!

# Bad data in = Bad results out (Garbage In, Garbage Out)

2Ô∏è‚É£ Data Cleaning Checklist

Cleaning involves making sure the data is accurate, consistent, and complete.

  • Handle Missing Values: Decide whether to **impute** (fill with mean/median/mode) or **drop** rows/columns with missing data.
  • Outlier Management: Identify extreme values (outliers) that could skew your model and decide whether to remove or transform them.
  • Data Type Conversion: Ensure columns are the correct type (e.g., numbers aren't stored as strings).
  • Deduplication: Remove duplicate rows that might inflate your dataset size artificially.

3Ô∏è‚É£ Data Wrangling with Pandas

**Pandas** is the essential Python library for structured data manipulation. It introduces the `DataFrame` object, which acts like a powerful spreadsheet.

Basic Pandas operations:

import pandas as pd

# 1. Read Data
df = pd.read_csv('intern_data.csv')

# 2. Check info
print(df.info())

# 3. Handle Missing Values (Impute with Mean)
df['Age'].fillna(df['Age'].mean(), inplace=True)

Page 3: Exploratory Data Analysis (EDA) & Visualization

1Ô∏è‚É£ What is EDA?

Exploratory Data Analysis (EDA) is the initial investigation of data to discover patterns, detect anomalies, test hypotheses, and check assumptions, often with the help of statistical graphics and visualization. It guides your entire modeling strategy.

2Ô∏è‚É£ Key Python Visualization Tools

We use Python libraries for plotting:

  • Matplotlib: The fundamental plotting library in Python. It's powerful but often verbose.
  • Seaborn: Built on top of Matplotlib, it provides a high-level interface for drawing attractive and informative statistical graphics.

3Ô∏è‚É£ Essential Plot Types

Plot Type Purpose Primary Insight
Histogram Shows the distribution of a single numerical variable. Checking for normality, skewness.
Box Plot Displays spread (quartiles) and identifies outliers. Finding extreme values and distribution range.
Scatter Plot Shows the relationship between two variables. Checking for correlation (positive, negative, none).
Heatmap Visualizes matrices, often correlation matrices. Quickly seeing which features are correlated with each other.

4Ô∏è‚É£ A Simple Scatter Plot Example

Page 4: Introduction to Machine Learning Modeling

1Ô∏è‚É£ Supervised Learning [Image of Supervised vs Unsupervised Machine Learning]

Supervised learning involves training a model on **labeled data** (input features $X$ and known output labels $y$). The goal is to predict the output based on new input data.

  • Classification: Predicts a **category** (e.g., Spam or Not Spam, Yes or No, Dog or Cat).
  • Regression: Predicts a **continuous numerical value** (e.g., House Price, Stock Value, Temperature).

2Ô∏è‚É£ The Machine Learning Workflow (Scikit-learn)

Scikit-learn is the most popular library for classical ML in Python. The universal workflow involves these steps:

from sklearn.model_selection import train_test_split
from sklearn.linear_model import LogisticRegression

# 1. Separate Features (X) and Target (y)

# 2. Split Data (Most critical step!)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# 3. Choose and Train Model
model = LogisticRegression()
model.fit(X_train, y_train)

# 4. Predict
predictions = model.predict(X_test)

3Ô∏è‚É£ Key Learning Concepts

Before fitting a model, you must split your data to ensure the model isn't tested on data it was trained on. This prevents a common problem known as **Overfitting**.

  • Training Set: Used to build the model. (e.g., 70% of data)
  • Test Set: Used to evaluate the model's performance on unseen data. (e.g., 30% of data)

Page 5: Model Evaluation & Practical Takeaways

1Ô∏è‚É£ Classification Evaluation Metrics

Accuracy isn't enough! We use several metrics for Classification models:

  • Accuracy: Overall number of correct predictions / Total predictions. (Simplest measure)
  • Precision: Of all the cases predicted as positive, how many were actually positive? (Important when False Positives are costly)
  • Recall (Sensitivity): Of all the cases that were actually positive, how many did the model correctly identify? (Important when False Negatives are costly, e.g., disease detection)

2Ô∏è‚É£ Dealing with Model Flaws

Two common problems:

  • Overfitting: Model learns the training data and noise too well. It fails completely on new data. (Solution: Use simpler models, get more data, or use regularization.)
  • Underfitting: Model is too simple and cannot capture the underlying pattern of the data, performing poorly everywhere. (Solution: Use a more complex model or add more relevant features.)

Key Takeaways from This Course

  • Data Science combines **Stats, Computer Science, and Domain Knowledge** to extract value.
  • You will spend **80% of your time on Data Cleaning** using tools like **Pandas**.
  • **EDA** and **Visualization** (Matplotlib/Seaborn) are necessary to guide your modeling decisions.
  • Machine Learning tasks are either **Regression** (predicting a number) or **Classification** (predicting a category).
  • Model evaluation must go beyond accuracy, focusing on **Precision** and **Recall** based on the business problem.
